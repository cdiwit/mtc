# MTC 列表搜索与搜索历史设计

## 背景

原始方向尝试为主界面增加树视图，用工作目录进行层级分组。但在实际界面验证中，树视图的布局和交互复杂度明显高于预期，用户当前的真实优先级也发生了变化：相比树形浏览，更希望保留稳定的列表视图，并把搜索能力做得更顺手。

本次设计将目标收缩为：

1. 保留单一列表视图
2. 保留并强化列表搜索
3. 增加可持久化的搜索历史
4. 支持从历史中快速复用搜索词
5. 支持删除单条历史与清空全部历史

## 用户已确认的需求

- 可以不要树视图
- 有一个搜索功能就很好
- 搜索结果希望能进入历史搜索
- 搜索历史需要可删除
- 搜索历史展示形式采用“搜索框旁边历史按钮，下拉显示历史”
- 搜索历史需要持久保存
- 只有按 Enter 才把关键词写入历史

## 推荐方案

采用“单一列表视图 + 搜索框 + 历史按钮”的方案。

### 选择原因

- 界面最稳定，直接消除树视图相关的布局和切换问题
- 可最大化复用当前已经做好的列表过滤逻辑
- 搜索历史按钮比复杂的输入框内嵌下拉更容易稳定实现
- 更符合当前用户优先级：稳定、顺手、减少界面复杂度

## 界面设计

### 顶部搜索区

右侧顶部保留搜索输入框，并在输入框右侧增加“历史”按钮。

交互方式：

- 输入关键词
- 按 Enter 执行搜索并写入历史
- 点击“历史”按钮展开历史菜单
- 点击历史项后，回填搜索框并立即执行搜索
- 历史菜单支持删除单条历史和清空全部历史

### 主内容区

主内容区仅保留现有列表视图：

- 名称
- 描述
- 工作目录
- 终端

不再保留树视图和列表/树模式切换。

## 搜索历史数据结构

搜索历史作为程序设置的一部分，挂在 `AppSettings` 下：

```cpp
std::vector<std::string> searchHistory;
```

### 保存位置

搜索历史和其他设置一起存放在 `data/config.json` 中，通过现有配置持久化机制读写。

推荐放入：

- `AppConfig.settings.searchHistory`

这样可以复用现有 `ConfigManager` 的配置加载与保存路径。

## 搜索历史规则

### 写入时机

只有在用户按 Enter 明确执行搜索时，才把关键词写入搜索历史。

### 写入规则

- 空字符串不写入
- 自动对关键词做首尾空白去除
- 重复关键词不新增，而是移动到最前面
- 只保留最近 10 条历史记录

### 历史复用规则

点击历史项时：

1. 把该关键词回填到搜索框
2. 立即执行搜索
3. 把该关键词移动到历史最前面

### 删除规则

支持两种删除方式：

- 删除单条历史
- 清空全部历史

单条删除时，从历史数组移除指定关键词并立即保存。

清空全部时，清空历史数组并立即保存。

## 历史菜单交互

点击“历史”按钮后：

- 如果有历史：列出最近搜索词，并提供“删除某条历史…”和“清空历史”
- 如果没有历史：显示“暂无搜索历史”

### 单条删除交互

为了降低实现复杂度，不使用“每一行右侧一个删除按钮”的复杂菜单项，而采用更稳定的两步交互：

1. 点击“删除某条历史…”
2. 弹出简单选择框，选择要删除的历史词
3. 删除后立即刷新历史菜单内容

## 代码结构建议

### 保留

保留以下已实现部分：

- `ProfileTreeBuilder.cpp` 中与搜索相关的纯逻辑：
  - `NormalizeSearchText`
  - `MatchesSearch`
  - `FilterProfiles`
- `MainFrame` 中基于 `m_visibleProfiles` 的列表过滤链路
- 当前列表展示与列表选中逻辑

### 回退/删除

从主界面层移除以下内容：

- `m_treeView`
- 列表/树模式切换控件
- 树视图刷新逻辑
- 树节点数据绑定逻辑
- 树视图选择与激活逻辑
- 树视图相关状态恢复逻辑

### 新增

#### `MainFrame`

新增：

- 历史按钮控件，例如 `wxButton* m_btnSearchHistory;`
- 搜索回车处理
- 历史菜单弹出逻辑
- 历史项点击后的搜索执行逻辑
- 删除单条历史/清空全部历史的交互逻辑

#### `Types.h`

在 `AppSettings` 中增加：

```cpp
std::vector<std::string> searchHistory;
```

#### `ConfigManager`

扩展配置读写：

- 读取 `settings.searchHistory`
- 保存 `settings.searchHistory`
- 保持与现有设置更新逻辑一致

## 实现顺序

1. 在 `Types.h` 中加入 `searchHistory`
2. 在 `ConfigManager` 中接入 `searchHistory` 的读取与保存
3. 保留现有列表过滤逻辑，收口“按 Enter 才写入历史”
4. 在 `MainFrame` 中增加历史按钮
5. 实现历史菜单
6. 实现删除单条历史和清空历史
7. 移除树视图和模式切换相关 UI 与逻辑

## 异常与边界处理

- 空搜索词不写入历史
- 删除不存在的历史词时静默忽略
- 导入配置时若没有 `searchHistory` 字段，使用空历史
- 历史超过 10 条时自动截断旧记录
- 历史菜单无数据时显示“暂无搜索历史”

## 测试设计

### 逻辑测试

优先验证：

- 历史去重
- 历史移动到最前面
- 历史最大条数限制
- 删除单条历史
- 清空全部历史
- 配置文件中的 `searchHistory` 正确读写

### 手工 UI 验证

至少覆盖以下场景：

1. 输入关键词并按 Enter 后，列表过滤生效
2. Enter 后历史新增并持久保存
3. 点击历史项可回填并执行搜索
4. 重启程序后历史仍然存在
5. 删除某条历史后菜单更新
6. 清空历史后显示“暂无搜索历史”

## 实施范围控制

本次设计只解决：

- 列表搜索
- 搜索历史
- 搜索历史的持久化和删除
- 主界面从双模式收缩为单列表模式

本次明确不包含：

- 树视图修复和继续演进
- 多级筛选器
- 标签系统
- 收藏夹/最近使用配置系统
- 搜索高亮或复杂搜索语法

这样可以让功能集中、稳定，并尽快交付到可用状态。
