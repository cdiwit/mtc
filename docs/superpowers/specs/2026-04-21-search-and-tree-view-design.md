# MTC 搜索与树形视图设计

## 背景

当前主界面使用平铺列表展示全部配置。当配置数量增多后，用户很难快速定位目标配置，也无法按工作目录形成稳定的浏览结构。

本次设计目标是：

1. 增加搜索能力，帮助用户快速定位配置
2. 增加按工作目录分组的树形视图
3. 保留现有列表视图，提供列表与树形双模式切换
4. 在不重构整个 UI 框架的前提下，以最小风险落地

## 用户已确认的需求

- 采用双模式并存，而不是只保留一种主视图
- 搜索默认匹配：名称、描述、工作目录
- 树形模式按工作目录路径层级展开，而不是按完整目录字符串单节点分组
- 树形模式下搜索后保留树结构，并自动展开到命中项
- 主界面布局采用“左侧轻导航 + 右侧内容区”的 B 方案
- 列表模式保持简单，左侧只负责视图切换，不增加复杂筛选导航

## 推荐实现方案

采用方案 1：保留现有 `wxListView` 作为列表模式，新增 `wxTreeCtrl` 作为树形模式，在右侧内容区切换显示。

### 选择原因

- 与现有 `MainFrame` 结构最贴近，改动最小
- 风险低，便于在当前代码基础上增量实现
- 不需要一次性引入更复杂的统一数据视图控件体系
- 可以先解决查找与分组问题，避免超出本次范围的 UI 重构

### 不采用的方案

#### 统一改为 `wxDataViewCtrl` / `wxDataViewTreeCtrl`

不采用，原因是实现复杂度明显更高，与当前项目直接手写控件和刷新逻辑的风格不匹配。

#### 使用“伪树列表”模拟树形

不采用，原因是展开/折叠、搜索保留树结构、节点选择等行为会变脆，长期维护成本高。

## 界面设计

### 总体布局

主窗口调整为“左侧导航与操作区 + 右侧搜索与内容区”。

#### 左侧区域

左侧区域从上到下包含：

1. 视图模式切换
   - 列表模式
   - 树形模式
2. 当前结果统计
3. 现有操作按钮
   - 新建配置
   - 编辑
   - 删除
   - 启动终端
   - 复制
   - 导入
   - 导出

视图切换建议使用 `wxRadioButton` 实现，语义明确，也最符合当前需求中的“轻导航感”。

#### 右侧区域

右侧区域从上到下包含：

1. 始终可见的搜索框
2. 内容区
   - 列表模式下显示 `wxListView`
   - 树形模式下显示 `wxTreeCtrl`

搜索框不随模式切换而隐藏，避免用户认知跳变。

### 模式行为

#### 列表模式

- 仅显示经过搜索过滤后的 profile 平铺结果
- 不做目录分组
- 双击条目启动终端
- 作为“快速查找结果视图”使用

#### 树形模式

- 按 `workingDirectory` 路径层级生成树
- 叶子节点为 profile
- 无工作目录的 profile 放入固定分组 `未设置工作目录`
- 双击目录节点展开/折叠
- 双击 profile 叶子节点启动终端

## 搜索设计

### 搜索范围

默认匹配以下字段：

- `profile.name`
- `profile.description`
- `profile.workingDirectory`

### 搜索规则

- 大小写不敏感
- 子串匹配
- 自动去除首尾空白
- 空搜索词表示不过滤，显示全部数据
- 中文与英文都按普通文本匹配

### 列表模式中的搜索结果

列表模式下，搜索结果直接表现为过滤后的平铺 profile 列表。

### 树形模式中的搜索结果

树形模式下，不是先渲染整棵树再高亮，而是采用以下规则：

1. 先对全部 profile 按搜索条件过滤
2. 使用命中的 profile 重新构建树
3. 只保留命中 profile 所在的目录路径
4. 自动展开所有祖先节点，使命中项直接可见

这样可以同时保留路径上下文和查找效率。

## 树形数据模型

为了避免把路径分组和树构建逻辑堆在 `MainFrame` 中，引入一个轻量中间结构：

```cpp
struct ProfileTreeNode {
    std::string label;
    std::string fullPath;
    std::vector<ProfileTreeNode> children;
    std::vector<const Profile*> profiles;
};
```

### 字段含义

- `label`：当前节点显示名
- `fullPath`：当前节点对应的完整路径，用于唯一识别
- `children`：子目录节点
- `profiles`：挂在当前目录节点下的 profile

### 路径分组规则

Windows 示例：

- `D:/Work/Proj/A` 拆分为 `D:` → `Work` → `Proj` → `A`

Unix 示例：

- `/home/user/app` 拆分为 `/` → `home` → `user` → `app`

每个 profile 插入到其 `workingDirectory` 对应的最终目录节点下。多个 profile 指向同一目录时，共享目录节点，在该节点下显示多个叶子项。

无工作目录的 profile 放入固定分组 `未设置工作目录`。

## 交互设计

### 选中行为

引入统一的“当前选中的 profile”状态，而不是让按钮逻辑直接依赖某一个具体控件。

#### 列表模式

- 选中行表示选中一个 profile
- 双击行启动终端

#### 树形模式

- 选中目录节点时，不允许编辑、删除、启动、复制
- 选中 profile 叶子节点时，允许编辑、删除、启动、复制
- 双击 profile 叶子节点启动终端
- 双击目录节点展开或折叠

### 按钮状态

- 新建配置：始终可用
- 编辑：仅选中 profile 时可用
- 删除：仅选中 profile 时可用
- 启动终端：仅选中 profile 时可用
- 复制：仅选中 profile 时可用
- 导入：始终可用
- 导出：存在至少一个 profile 时可用

### 切换视图时的上下文保留

模式切换时遵循以下规则：

- 保留当前搜索词
- 如果当前选中的 profile 在新视图中仍存在，则继续选中
- 如果找不到，则清空选中状态
- 从列表切换到树形时，自动展开到当前 profile 所在路径
- 从树形切换到列表时，若当前选中的是 profile，则在列表中选中对应条目；若当前选中的是目录节点，则不保留选中

### 数据变更后的刷新策略

所有会变更 profile 集合的操作最终都走统一刷新入口。

建议将当前单一的 `RefreshProfileList()` 重构为以下职责分离的接口：

- `RefreshView()`
- `RefreshListView(filteredProfiles)`
- `RefreshTreeView(filteredProfiles)`

刷新后的行为要求：

- 新建成功后选中新建的 profile
- 编辑成功后保持选中该 profile
- 复制成功后选中新副本
- 删除成功后优先选中后一个同级项，否则选前一个，没有则清空选中
- 导入成功后保留当前搜索词，并在当前模式下重新过滤和刷新

## 代码结构建议

本次改动主要集中在 `MainFrame` 层，不修改 `ConfigManager` 与 `TerminalLauncher` 的公开接口。

### 预计主要修改文件

- `src/ui/MainFrame.h`
- `src/ui/MainFrame.cpp`

### 建议新增的辅助状态和控件

在 `MainFrame` 中新增：

- 搜索输入控件
- 列表模式单选按钮
- 树形模式单选按钮
- 树控件
- 当前视图模式状态
- 当前搜索词状态
- 当前选中 profile 标识

### 建议新增的辅助函数

可以先放在 `MainFrame.cpp` 内部静态函数中，若后续变复杂，再提取到独立文件：

- `NormalizeSearchText(...)`
- `MatchesSearch(const Profile&, const std::string&)`
- `FilterProfiles(const std::vector<Profile>&, const std::string&)`
- `SplitWorkingDirectory(const std::string&)`
- `BuildProfileTree(const std::vector<const Profile*>&)`

## 异常与边界处理

只处理本功能中真实会发生的边界情况：

- 工作目录为空：归入 `未设置工作目录`
- 工作目录格式不规范：按字符串尽力分段，不依赖真实文件系统存在
- 搜索词为空：显示全部数据
- 搜索无结果：
  - 列表模式显示空结果
  - 树形模式显示空树或“无匹配结果”提示
- 切换视图后选中对象不存在：清空选中并更新按钮状态

本次不把“工作目录是否真实存在”作为前置校验条件，因为该功能是配置管理，不是文件系统校验。

## 测试设计

### 逻辑测试

优先测试最容易出错的纯逻辑：

- `MatchesSearch`
  - 空词
  - 英文大小写
  - 中文匹配
  - 工作目录命中
- `SplitWorkingDirectory`
  - Windows 路径
  - Unix 路径
  - 空路径
  - 带尾部分隔符路径
- `BuildProfileTree`
  - 同目录多个 profile
  - 多层目录
  - 无工作目录分组
  - 搜索后仅保留命中路径

### 手工 UI 验证

至少覆盖以下场景：

1. 列表模式输入关键词后即时过滤
2. 树形模式下搜索后保留树结构并自动展开
3. 选中目录节点时编辑、删除、启动按钮禁用
4. 选中 profile 节点时按钮恢复可用
5. 双击树形叶子节点可以启动终端
6. 新建、编辑、复制、删除后选中状态符合预期
7. 导入配置后仍保留当前搜索条件

## 实施范围控制

本次设计只解决以下问题：

- 大量配置下的快速查找
- 按工作目录层级浏览配置
- 在现有主界面中提供列表与树形双模式

本次明确不包含：

- 排序系统重构
- 多字段高级筛选
- 标签系统
- 收藏夹或最近使用视图
- 统一迁移到新的数据视图控件体系

这样可以保证实现范围聚焦，风险可控。
